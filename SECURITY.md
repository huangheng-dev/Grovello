# Security Policy

Grovello has not issued a stable release. Security fixes currently target the latest development branch.

Do not open public issues for vulnerabilities or exposed credentials. Before the first public release, enable GitHub private vulnerability reporting or publish a dedicated security contact.

Priority areas include tenant isolation, authorization, secret handling, connector permissions, prompt injection, tool escalation, consent and suppression, approval bypass, file processing, webhook verification, audit integrity, and model data policy.
