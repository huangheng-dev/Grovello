# 能力状态

| 状态 | 含义 |
| --- | --- |
| Foundation | 已有页面、产品定义、共享对象、架构边界与初始契约 |
| Simulated | 使用明确标注的种子或 Mock 数据，不产生外部能力声称 |
| Connected | 某供应商账号已通过契约、认证、健康、策略与沙箱测试 |
| Operational | 已验证端到端执行、失败处理、审计、衡量和运维手册 |
| Experimental | 通过显式开关使用，已声明限制与评估标准 |
| Planned | 已接受的架构项，但没有生产实现声明 |

当前仓库已通过阶段 0 验证。阶段 1“组织与访问控制”正在实现基础能力：已经建立组织、工作区、用户、团队、成员关系、角色、权限、策略、会话和审计规范模型；PostgreSQL 迁移启用工作区 RLS；种子访问契约覆盖已认证、未授权、跨租户、审计和恢复计划路径。开发身份 Header 在生产环境中禁用，OIDC 供应商和持久化访问目录尚未连接。外部渠道、分析投影和生产供应商账号也尚未连接；界面和访问目录使用明确标注的种子数据。
