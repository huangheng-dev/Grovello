# 能力状态

| 状态 | 含义 |
| --- | --- |
| Foundation | 已有页面、产品定义、共享对象、架构边界与初始契约 |
| Simulated | 使用明确标注的种子或 Mock 数据，不产生外部能力声称 |
| Connected | 某供应商账号已通过契约、认证、健康、策略与沙箱测试 |
| Operational | 已验证端到端执行、失败处理、审计、衡量和运维手册 |
| Experimental | 通过显式开关使用，已声明限制与评估标准 |
| Planned | 已接受的架构项，但没有生产实现声明 |

当前仓库已通过阶段 0 验证。阶段 1“组织与访问控制”正在实现基础能力：已经建立组织、工作区、用户、团队、成员关系、角色、权限、策略、会话和审计规范模型；PostgreSQL 迁移启用工作区 RLS；种子访问契约覆盖已认证、未授权、跨租户、审计和恢复计划路径。阶段 2“共享业务事实”现已具备第一条基础纵向切片：规范对象 ID、不可变版本、精确证据引用、资料完整性校验、幂等写入、审计血缘、事务 Outbox 事件和工作区 RLS。契约覆盖品牌、产品、商业方案、价格表、市场、ICP、证据、知识、资产与案例；Northstar 数据仍是明确标注的虚构种子。“品牌与市场”六个页面现已通过服务端 BFF 提供统一资料读取、受治理对象创建和不可变版本更新，并覆盖加载、空状态、不可用、未授权、冲突和校验失败。“业务与品牌”“产品、方案与定价”“市场与本地化”“理想客户画像”“证据”和“知识文档”现已提供双语专属表单；商业方案引用统一产品 ID，价格表可引用统一商业方案 ID 并保存带数字金额的价格条目，ICP 引用统一市场 ID 和可选产品 ID，并结构化保存购买委员会、资格条件与排除条件。证据记录包含来源血缘、核验状态、可靠性、使用权限、适用范围、关键发现和限制条件；知识文档可关联一个统一业务对象，并保存受治理正文、负责人、主题、受众和检索关键词。客户案例现可记录披露状态、获准客户名称、统一市场/产品/方案/ICP 引用、项目周期、挑战、方法、结构化衡量结果、限制条件、获准用途、授权依据和负责人；生效案例必须引用至少一个精确且不可变的证据版本。版本编辑器可将受支持的宣称固定引用到精确的不可变证据版本并记录结构化定位；知识页面可筛选从实时工作区 API 加载的记录。编辑时会保留旧版本中的未知附加属性。素材库的存储基础现已定义工作区隔离的上传会话、经核验的不可变 Blob、精确素材版本文件绑定、工作区 RLS 和分风险级素材权限。本地参考 S3-compatible Profile 已通过固定版本 MinIO 镜像、私有且启用版本控制的 Bucket、最小权限应用账号、有界的供应商中立操作以及显式健康端点完成连接。上传会话 API 现已提供幂等创建、精确限制大小/类型/SHA-256 的预签名 POST、带权限的状态查询/完成/取消操作、审计与 Outbox 血缘，以及持久化 Temporal 校验工作流。校验会检查供应商元数据，并流式读取对象计算实际 SHA-256。供应商中立恶意文件扫描契约与按摘要固定的 ClamAV 本地参考服务现已接入，并通过有界 Temporal Activity 流式扫描已核验对象。干净会话只推进到 `ready_to_finalize`；感染对象会复制到工作区隔离的隔离区精确 Key，按版本从暂存区删除，并保存扫描证据。耗尽重试或不可重试的失败会关闭式失败，取消则使用 Temporal 原生取消和持久化会话状态。扫描阶段不会创建 Ready Asset、不可变 Blob 或素材版本绑定。受治理最终化现已通过幂等 Temporal Saga 接通：只提升干净会话到工作区隔离的不可变对象版本，在事务中提交规范 Asset、版本和文件绑定，精确清理暂存版本，并对提交失败安全补偿。将素材设为 active 要求 `asset.approve`；受审计的短时私有下载要求 `asset.download`，且只接受精确的 active、clean、available 版本。素材库现已通过服务端 BFF 提供双语可运营基础界面：受治理浏览器上传、持久化状态追踪、干净会话最终化、规范目录读取、不可变版本历史和受审计安全下载授权均已接通，且不会向浏览器暴露对象 Key 或存储凭据。生产身份与通用审批工作流接通前，该能力仍标记为 `foundation`。持久化企业入驻、导入、语义检索、知识切块流水线、完整审批工作流和供应商同步同样尚未达到可运营状态；知识片段仍是未来流水线生成的派生对象，不允许所有者手工伪造。BFF 仅在显式本地开关下接受开发身份，API 生产环境仍拒绝开发 Header；OIDC 供应商和持久化访问目录尚未连接，外部渠道、分析投影和生产供应商账号也尚未连接。

P2-D1 至 P2-D4 现已为持久化工作区入驻和结构化业务事实导入提供经过本地验证的基础能力：带强制 RLS 的租户执行表、9 项分风险权限、幂等入驻/导入 API、受约束私有来源上传、专用 Temporal 完整性核验/扫描/校验/apply/补偿工作流、有界 UTF-8 CSV 与版本化 Grovello JSON 解析、不可变白名单映射、确定性规范校验、仅精确匹配的去重、脱敏审查证据、不可变 Dry-run Change Set、绑定策略版本的审批交接、规范业务事实写入、并发安全的补偿版本，以及带 `BusinessProfileActivated` 血缘的精确激活快照。P2-D4 通过薄 BFF 新增双语“企业资料配置”和“数据导入”操作路由、持久化进度与恢复状态、可访问表单和响应式布局。7 月 23 日的本地验收使用 PostgreSQL、Temporal、私有且启用版本控制的 MinIO Bucket、最小权限应用账号和 ClamAV，对浏览器上传的 CSV 完成上传、核验、扫描、映射、校验、计划、应用与补偿；该任务产生 9 条审计记录和 9 条 Outbox 事件。另一次全新工作区集成运行验证了精确激活和补偿。在生产身份、通用审批工作流、生产运维手册和非开发部署证据接通前，该能力仍标记为 `foundation`，不能标记为 `operational`。

P2-E1 现已提供经过本地验证的企业知识入库契约基础。迁移 `0012` 新增 5 张带强制 RLS 的工作区执行表、7 项分风险知识权限、不可变精确来源快照、pending Generation，以及为后续切片预留的检索 Receipt 存储。版本化 API 支持为符合条件的 active 来源版本幂等创建、列出和读取入库；只有内部服务 Actor 可以创建带精确来源与 Generation 血缘的规范不可变 `knowledge_chunk` 版本，通用所有者/导入 Mutation 和通用业务事实读取均不能绕过该边界。验证覆盖真实 PostgreSQL 迁移一致性、非超级用户跨工作区 RLS 隔离、HTTP 授权/幂等/冲突行为、流水线重放安全，以及精确审计与 Outbox 证据。P2-E1 仍是 `foundation`：解析、Temporal 提取、确定性切块、Embedding、词法或语义检索、引用和操作界面均未实现。

## 导航能力状态

导航能力目录保留全部十个产品业务域及未来能力的稳定身份。主侧栏只显示具备真实使用体验的 `active` 能力，以及明确说明实现边界的 `foundation` 能力。`reserved` 能力会在所属业务域的能力地图中保留交付阶段和目标用户，但不会作为可点击的主导航功能，也不声称已经可运营。系统架构归入帮助内容，不作为业务模块；高级自动化控制为管理员或开发者体验预留。

英文导航采用全球 B2B SaaS 与营收运营中通行的术语；简体中文按业务含义本地化，不逐字硬译。例如 Strategy & Plans 对应“战略与计划”，Quote-to-Cash 对应“报价到回款”，Unified Data 对应“统一数据”，AI Search (GEO) 对应“AI 搜索优化（GEO）”。
