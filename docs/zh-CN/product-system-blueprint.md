# Grovello 完整产品系统蓝图

本蓝图在功能实现以前固定目标产品架构，完整定义闭环、职责、契约、状态、治理、恢复和前端入口。实现可以分阶段进行，但不能静默缩小目标架构。`AGENTS.md` 仍然是最高级开发规则。

## 1. 架构闭合原则

Grovello 只有一个产品、一套规范业务模型、一个持久化工作流平面、多个有边界的 Agent，以及版本化供应商连接器。

```text
企业增长操作系统
├─ 共享业务事实
├─ 增长指挥与策略治理
├─ 专业 Agent 推理
├─ 持久化工作流执行
├─ 供应商中立连接器
├─ 客户与营收运营
└─ 结果、归因、实验和学习反馈
```

第一条黄金闭环是“全球 B2B 增长”。默认公开验收数据采用虚构的 **Northstar Industrial（北辰工业）**工作区：一家工业自动化供应商评估并进入德国 B2B 市场。企业、行业、来源国、目标市场、语言、币种和渠道都必须是可替换数据。

| 数据与智能 | 身份图、事件分类、指标、归因、报表、实验、市场情报 | 全部领域事件和外部信号 | 可信证据、洞察和建议 | 数据中心、归因、报表、实验、情报 |

## 3. 黄金闭环工作流图

| 持久工作流 | 触发 | 必需状态 | 成功事件 | 补偿或恢复 |
| --- | --- | --- | --- | --- |
| 企业入驻 | 创建工作区 | 品牌、市场和策略草稿 | `BusinessProfileActivated` | 保留草稿并报告校验缺口 |
| 市场情报 | 选择市场或定时刷新 | 来源、时效策略、ICP | `MarketBriefPublished` | 标记过期/部分证据，绝不编造缺口 |
| 策略规划 | 目标、预算或重要信号 | 证据、约束和预测模型 | `GrowthPlanApproved` | 修改、拒绝、过期或替换计划 |
| 资产生产 | 已审批活动 Brief | 主张、引用和渠道规格 | `AssetVariantApproved` | 重新生成、隔离无证据主张、恢复旧版本 |
| 渠道执行 | 已审批资产或序列 | 连接器健康、授权、预算和同意 | `TouchpointRecorded` | 安全重试、暂停账号、取消排期、对账回执 |
| 潜客情报 | 访客、发现企业或导入 | 来源、目的、身份和法律依据 | `LeadQualified` 或 `LeadRejected` | 合并重复、抑制、删除或请求审核 |
| 会话与会议 | 入站消息或已审批开发 | 联系人、渠道、分配和回复策略 | `MeetingBooked` 或 `ConversationResolved` | 停止序列、执行退订、人工接管、重新打开 |
| 商机管理 | 合格潜客或会议 | 企业、负责人、阶段规则和证据 | `OpportunityWon` 或 `OpportunityLost` | 审计后重开、重新分配、超时关闭并保留原因 |
| 从订单到回款 | 接受商务方案 | 审批、价格版本、客户和税务上下文 | `PaymentReconciled` 与 `RevenueRecognized` | 作废、贷项、更正、退款或升级差异 |
| 客户增长 | 赢单或订单生效 | 成功负责人、里程碑和权益 | `RenewalWon`、`ExpansionWon` 或 `ReferralCreated` | 风险方案、升级、退出和原因记录 |
| 归因与学习 | 结果产生或评估窗口关闭 | 事件、成本、收入和模型版本 | `LearningEvidencePublished` | 重算投影、暴露不确定性并保留旧结果 |
| 下一轮决策 | 新证据或规划周期 | 目标状态、学习和约束 | `StrategyDecisionRecorded` | 接受、编辑、拒绝、暂停或回滚 |

Temporal 负责这些长期状态、定时器、重试、取消、补偿、信号和审批。LangGraph 可以在有边界的 Activity 中调用，但绝不能成为商务事实或工作流事实账本。

## 4. Agent 拓扑

Agent 按专业能力分工；系统不存在拥有一切权限的“万能中央 Agent”。增长决策 Agent 可以组织建议，但没有工作流与策略授权就不能执行。

| Agent 组 | 职责 | 强制边界 |
| --- | --- | --- |
| 市场情报 | 研究市场、需求、竞品、活动、企业和意向 | 保存来源、时效和置信度；区分事实与推断 |
| 策略与规划 | 目标、情景、渠道组合、预算建议和作战计划 | 声明假设、预期影响、不确定性和审批需要 |
| 品牌守卫 | 检查术语、语气、主张、禁区、证据和本地化 | 拒绝无证据主张；不得编造认证或客户案例 |
| 内容 | 生成 Brief、文案、图片和渠道变体 | 只使用已批准事实；保留变体及 Prompt/模型血缘 |
| SEO | 搜索意图、关键词、集群、页面、内链和技术建议 | 优化合格业务结果，不追求无边界页面数量 |
| GEO | 问题、实体、主张、引用、答案可见度和引荐质量 | 保留证据和来源质量；不能把 GEO 与 SEO 混为一体 |
| 视频 | 脚本、分镜、配音、字幕、素材和渲染计划 | 发布前检查版权、肖像、披露和审批 |
| 社交与发布 | 内容日历、渠道适配、发布和回复建议 | 遵守连接器策略、频率、身份和审核规则 |
| 广告 | 受众、素材、实验和预算建议 | 预算上限、归因、禁用受众和风险阈值人工审批 |
| 潜客研究与补全 | 企业发现、联系人补全、验证和评分 | 来源、法律依据、去重、准确性和抑制检查 |
| 客户开发 | 个性化、序列、时机和下一步建议 | 同意/法律依据、退订、频率、送达率和品牌策略 |
| 会话 | 分类、拟回复、分配、预约和升级 | 不得冒充真人；按工作区策略披露或请求审核 |
| 销售 | 会议准备、资格判断、异议和下一步 | 人工掌握商务权力；不得编造条款或承诺 |
| 商务文档 | 方案和报价草稿、合同数据组装 | 使用已批准价格/版本、权限、法律模板和签署边界 |
| 客户成功 | 入驻、里程碑、健康、风险和增购建议 | 客户上下文、权益、服务策略和人工升级 |
| 归因与实验 | 归因、评估、异常和学习总结 | 模型版本、不确定性、数据质量和可复现性 |
| 合规与风险 | 评估策略、敏感数据、同意、抑制和工具风险 | 可以阻止执行；其他 Agent 不能绕过 |
| 评测 | 离线/在线质量、回归、安全、成本和结果检查 | 独立数据集和版本化结果通过后才能升级 |

每个 Agent 都必须拥有版本化输入输出 Schema、允许工具、模型策略、Token/成本/时间上限、评测门禁、人工中断规则和审计血缘。

## 5. 连接器架构

连接器注册中心有意保持广泛并与供应商解耦，完整覆盖：

| 类型 | 能力 |
| --- | --- |
| 自有网站 | 页面发布/更新/下线、表单、Webhook、Sitemap、转化事件 |
| 搜索与答案发现 | 搜索表现、索引、关键词/问题数据、AI 答案证据 |
| 社交与社区 | 在授权范围内发布、评论/回复、收件箱、账号健康和分析 |
| 广告 | 活动、受众、素材、预算、投放、转化和成本同步 |
| 邮件 | SMTP/API 发送、邮箱/回复同步、退信、投诉、退订和送达率 |
| 即时通信 | 已批准企业消息的收发、模板、身份和退订 |
| CRM 与日历 | 企业/联系人/商机同步、活动、会议、空闲时间和预约 |
| 企业与联系人数据 | 搜索、补全、验证、来源和置信度 |
| 电商与财务 | 产品、订单、发票、付款、退款、对账和会计事件 |
| 分析与数据 | 事件采集、导入导出、数仓、BI 和归因数据 |
| 存储与知识 | 文档、资产、权限、修订、检索和引用 |
| 模型与媒体 | 文本、结构化输出、Embedding、图片、音频、视频和审核 |

每个连接器必须声明 Manifest 版本、供应商、账号、能力、配置 Schema、密钥引用、Scope、外连域名、风险、法律/数据说明、限流、幂等行为、健康、Webhook、沙箱行为、结果 Schema、可逆性和能力状态。

供应商接入顺序固定为：官方 API、Webhook、标准协议、授权导入导出，最后才是受控浏览器降级。浏览器自动化不得绕过平台控制。安装连接器只授予配置能力，不自动授予执行权。

第一套参考集成组合在架构上已经固定：Grovello 自有页面与事件采集、标准邮件发送/回复、日历预约适配器、通用签名 Webhook、原生 CRM/商机记录、原生商务记录，以及手工或连接式付款对账。供应商专属的社交、广告、搜索、消息、CRM 和财务适配器统一接入相同契约。真实生产账号属于部署输入，不是架构缺口。

## 6. 数据与事件主干

PostgreSQL 是事务事实源。统一身份图连接匿名访客、企业、联系人、潜客、会话、商机、客户、订单和收入身份。合并、拆分、别名和删除操作必须审计，并在策略允许时可逆。

最小事件主干为：

```text
SignalObserved
→ StrategyProposed / StrategyDecisionRecorded
→ AssetCreated / AssetApproved
→ PublicationDelivered / AdDeliveryObserved / MessageDelivered
→ VisitObserved / AccountIdentified / LeadQualified
→ ReplyReceived / MeetingBooked / OpportunityCreated
→ QuoteAccepted / ContractExecuted / OrderConfirmed
→ InvoiceIssued / PaymentReconciled / RevenueRecognized
→ OnboardingMilestoneReached / HealthChanged
→ RenewalWon / ExpansionWon / ReferralCreated
→ AttributionCalculated / ExperimentEvaluated / LearningEvidencePublished
```

业务写入与 Outbox 事件在同一事务提交。消费者和工作流使用幂等键。投影携带 Schema/版本血缘并可重建。收入、毛利、付款、同意和合同事实绝不能来自 LLM。

## 7. 默认自主执行与审批策略

系统默认采用风险分级自主执行，不采用“全自动”或“全部人工审核”两个极端。

| 等级 | 示例 | 默认权限 |
| --- | --- | --- |
| R0 读取/分析 | 检索、分类、预测、内部报表 | 在访问策略内自动执行 |
| R1 草稿/内部 | 内容草稿、计划、回复草稿、报表、内部任务 | 自动创建，人工可编辑或删除 |
| R2 可逆外部动作 | 排期发布、小批量已批准开发、沙箱变更 | 经过策略批准；首次和重大变更需要审核 |
| R3 商务/敏感动作 | 广告预算、群发、折扣、报价、合同、导出、身份合并 | 人工明确审批，并设置范围和证据 |
| R4 禁止动作 | 绕过控制、欺骗身份、违法采集、泄露密钥、编造收入、未授权付款 | 永远不能执行 |

工作区策略可以更严格。通过成功评测以后，可以在明确限制内放宽 R2，但永远不能放宽 R4。每次审批记录建议版本、证据、审批人、范围、过期时间和最终执行。

## 8. 前端信息架构

左侧十个一级业务域保持稳定。新增两个入口把已确认定位真正变成可操作功能：

- **增长闭环**位于“增长指挥”，用于查看黄金闭环阶段、就绪度、所有者输入、阻塞、Run 和验收证据。
- **市场与本地化**位于“品牌与市场”，用于设置市场、区域、语言、币种、法规、渠道可用性、商务惯例和市场进入假设。

每个运营页面使用与业务相符的主内容，同时遵守统一交互语法：

```text
页面身份与业务结果
→ 范围筛选及新建/导入/连接动作
→ 按需提供总览/记录/工作流/分析/设置
→ 状态、负责人、策略、审批和能力证据
→ 正常/空/加载/部分成功/失败/无权限状态
→ 审计、血缘和增长闭环反馈
```

经营总览负责汇总结果；增长闭环负责跨域进度；模块页面负责领域工作；Run 中心负责执行诊断；审批中心负责人类权力；数据与智能负责解释和学习。工作区与语言继续放在右上角个人菜单中；左侧导航分组默认展开、可折叠并记忆状态。

## 9. 可靠性与恢复闭合

- HTTP 接口只排队长任务，不同步执行采集、媒体、发布、批量开发或模型链。
- Temporal 提供重试、退避、超时、取消、补偿、暂停/继续、信号和版本管理。
- 外部写入使用幂等能力或本地去重账本，并保存供应商回执。
- 结果不明确的超时进入对账，不能盲目重试。
- 撤销连接器凭证不能删除业务历史。
- 活动部分成功必须按单项保留结果，并能选择性重试。
- Webhook 必须认证、去重、按需排序并可重放。
- Outbox 防止领域事件丢失；死信保留原因、Payload 引用、负责人和恢复动作。
- 备份、Schema 迁移、工作流版本和回滚都是发布门禁。
- 可观测性必须关联请求、Run、工作流、Agent、工具、连接器、成本和业务结果 ID。

## 10. “架构已完整”的定义

每一项承诺能力都对应以下内容时，才算前期架构完整：

1. 固定业务域和前端入口；
2. 规范输入与输出对象；
3. 命令/事件契约和持久工作流负责人；
4. 需要推理时使用有边界的 Agent 和允许工具；
5. 需要外部执行时具备连接器能力与真实供应商状态；
6. 授权、风险、同意、审批和审计规则；
7. 幂等、失败、取消、对账和补偿行为；
8. 可衡量结果、归因和下一轮反馈；
9. 能力状态与端到端验证证据；
10. 双语文档和迁移兼容性。

实现尚未完成的部分通过能力状态和路线图阶段追踪，不能变成未定义架构，也不能藏在一个通用 Agent 后面。

## 11. 仅需所有者提供的输入

产品架构已经为验收数据、自主执行、连接器契约和第一套参考集成组合提供默认答案。只剩一类事项需要所有者输入，但不构成架构缺口：

- 启用真实连接器时提供生产供应商账号、凭证、预算和已授权企业数据。

产品所有者已于 2026 年 7 月 19 日选择 `AGPL-3.0-only`。AI 可以准备配置和对比，但不能伪造访问权，也不能在没有批准时更改法律许可证。
